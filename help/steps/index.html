Filters Developer's Guide Okapi Framework

The default distribution of the framework comes with the pre-defined steps listed below. Note that some of them may be Beta or Alpha versions. See the documentation of each step for more information.

The letters between parenthesis indicates what type of data the step takes and what type it sends. RD=Raw Document, FE=Filter Events.

  • Batch Translation (RD > RD) - Creates translation memories using external tools.
  • Filter Events to Raw Document (FE > RD) - Merges filter events back into the original file format.
  • Format Conversion (FE > FE) - Converts extracted data to a given file format.
  • Full-Width Conversion (FE > FE) - Converts to or from full-width characters
  • Raw Document to Filter Events (RD > FE) - Extracts translatable data from a raw document and convert it into filter events.
  • Segmentation (FE > FE) - Segments extracted text.
  • TM Import (FE > FE) - Imports text units into a Pensieve TM.
  • Translation Comparison (FE > FE) - Compares the translations of the same entries between several files.

You may have additional steps available as plug-ins. For more information about those steps, look in the dropins directory of your installation.
