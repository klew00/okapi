Utility – Translation Comparison

Overview

This utility allows you to compare two sets of translated text.

The input is two document, the first one with the base translation, the second one with the translation to compare to the base translation. The two documents must have the same number of entries and the entries must be in the same order.

For each entry the utility compare the two translated text and calculate a score that indicates how similar the text of the second document is from the text of the first translation. the score is a number between 0 and 100. The score is based on the Levenstein distance and additional processing. You can find more information on that mertic at http://www.merriampark.com/ld.htm and http://en.wikipedia.org/wiki/Levenshtein_distance.

The utility outputs an HTML document for each pair of input document. That file lists the pairs of text and the resulting score. You can also generate a TMX document with the pairs and their scores.

Input Parameters

  • The list of the documents containing the base translation (Input list 1)
  • The list of the corresponding documents.containing the translation to compare to the base one (Input list 2)
  • The source language and default input encoding.
  • The target language and default input encoding.

Configuration

Output Tab

HTML Output

Generate output tables in HTML -- Set this option to create one HTML file for each pair of input document, where the results of the comparisons are listed. The name of the HTML result file is the name of the document to compare with the base document, plus a .html extension, (Configuration key = generateHTML).

Open the first HTML output after completion -- Set this option to open the first HTML result file created when the process is completed. (Configuration key = autoOpen).

TMX Output

Generate a TMX output document -- Set this option to create one single TMX document that contains all the translations compared and their scores, for all input documents. (Configuration key = generateTMX).

Enter the full path of the TMX document to generate. If one already exists it will be overwritten (you will be prompted). (Configuration key = tmxPath).

Suffix for the second target language code -- Enter the suffix to append to the target language code for the second target entries. For example, enter -mt to indicate that the second target language is a machine-translation generated text. If you do not enter a suffix, the language code for both target entries will be the same and the TMX document will include only the last one. (Configuration key = trgSuffix).
