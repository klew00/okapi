Okapi Framework - Utilities

XSL Transformation
Utilities Okapi Framework

This utility allows you to apply a given XSL Transform template to an input document and generate the resulting output.

This utility does not use filters.

Input Parameters

  • The list of the XML documents to transform (Input list 1).
  • The names and locations of the output documents.

Configuration

Options Tab

Path of the XSLT template to apply -- Enter the full path of the XSLT template to use to generate the output. You can use the variable ${ProjDir} to specify the directory of the project.

Parameters -- Enter the list of any required parameters for this template. the format must be name=value with one parameter per line. For example:

Lang=en
tmFile=myTM.tmx
inputFile=${Input2}

You can use the following variables in the parameters values:

  • ${SrcLang} = The code of the current source language.
  • ${TrgLang} = The code of the current target language.
  • ${Input1} = The full path of the input document of the Input list 1.
  • ${Output1} = The full path of the output document corresponding to the input document of the Input list 1.
  • ${Input2} = The full path of the input document of the Input list 2.
  • ${Output2} = The full path of the output document corresponding to the input document of the Input list 2.
  • ${Input3} = The full path of the input document of the Input list 3.
  • ${Output3} = The full path of the output document corresponding to the input document of the Input list 3.

Default Parameters -- Click this button to copy into the parameters list all the default parameters declared in the template.

Open Template -- Click this button to open the template file.

Use the transformer created by this factory class -- Set this option to use an XSLT transformer implementation different than the one your current Java settings gives. Enter the name of the factory class in the text field. The class specified must be in your Java class-path.

The default transformer class used depends on your implementation and version of Java. See the documentation of TransformerFactory.newInstance() for more details on how the lookup for the class is done.
