Okapi Framework - Utilities

Text Rewriting
Utilities Okapi Framework

This utility allows you to process an input document through a given filter and re-write it with different modifications (or no modifications). This is useful for example to perform a pseudo-translation and identify hard-coded text in some file formats.

Common Parameters

  • The list of the document to rewrite (Input list 1).
  • The source language.
  • The target language.
  • The default input encoding.
  • The output encoding.
  • The names and locations of the output documents.

Configuration

Options Tab

Type of change to perform -- Select a kind of change to apply. Several are available:

  • Keep the original text
  • Replace letters by Xs and digits by Ns.
  • Remove text but keep inline codes.
  • Replace text with translation resource matches.
  • Replace selected ASCII letters by extended characters.

Add the following prefix -- Set this option to add a prefix at the start of each text unit. Enter the text of the prefix.

Add the following suffix -- Set this option to add a suffix at the end of each text unit. Enter the text of the suffix.

Append the name of the item -- Set this option to add the name of each text unit at the end of its value. If the text unit has no name associated, the extraction ID is added instead.

Append the extraction ID of the item -- Set this option to add the extraction ID of each text unit at the end of its value. Extraction IDs are filter-specific.

Modify also the items with an existing translation -- Set this option to apply the changes also to text units that already have a translation (e.g. for multilingual files).

Marks segments with '[' and ']' delimiters -- Set this option to add delimiters to the segments in each text unit. The delimiters are just around the text (after the prefix if one is added, and before the item name, extraction ID, and suffix if they are added). If the Apply the following segmentation rules option is set you may have more than one segment per text unit, otherwise each text unit is a single segment.

Segmentation

Apply the following segmentation rules -- Set this option to segment the text units.

SRX file for the source -- Enter the full path of the SRX file to use for the source text. You can use the variable ${ProjDir} to specify the directory of the project. Click the Edit button to edit the rules.

SRX file for the target -- Enter the full path of the SRX file to use for the target text. You can use the variable ${ProjDir} to specify the directory of the project. Click the Edit button to edit the rules.

Translation Resource

Select the translation resource to use and, if needed, specify its settings. These parameters are enabled when you have selected the option Replace text with translation resource matches.

Settings -- Click this button to enter the settings for the selected translation resource. Not all translation resource connectors require settings. the current settings are displayed under the translation resource selection.

Be aware that some translation resources can be slow to access (especially those accessed through Internet), and using them may cause the process to be extremely long to complete.
