Utility – ID-Based Alignment

Overview

This utility allows you to align the source and target content of documents based on the identifiers of the text units. In addition, each text unit can be segmented and this utility provides a basic default alignment of the segments, along with a visual editor to fix any issue within those segments.

Warning:
This utility should be used only with documents formats that provide identifiers (e.g. Properties files). Text units without identifier will not be aligned.

Note:
The result of the alignment is generated in a single TMX document even when the input is made of several input documents.

Common Parameters

  • The list of the source documents to align.
  • The list of the corresponding target documents.
  • The source language and default input encoding.
  • The target language and default input encoding.

Configuration

Options Tab

-- Full path of the TMX document to generate (Configuration key = tmxPath)

Enter the full path of the TMX document where to put the results of the alignment. If the document already exists it will be overwritten.

-- Create a translation memory with the following path (Configuration key = createTM)

Set this option to create a translation memory from the results of the alignment. Enter the full path of the translation memory to create. If the translation memory already exists it will be overwritten.

-- Generate Trados workarounds (Configuration key = useTradosWorkarounds)

Set this option to generate Trados-specific output in the TMX document. This non-standard notation is needed to prevent loosing some characters such a the backslash or the curly braces when importing the TMX document in some versions of Trados Workbench.

TMX documents generated with this option are not standard TMX files and will likely not be imported correctly in tools correctly supporting TMX.

-- Segment the extracted text using the following SRX rules (Configuration key = segment)

Set this option to segment the content of each extracted text unit. If this option is set, the alignment is done first at the level of the text units (using their identifiers), then within each text unit, at the level of the segments. If this option is not set the alignment is done at the level of the text unit only.

-- SRX file for the source (Configuration key = sourceSrxPath)

Enter the path or URL of the SRX document where to take the SRX rules for the source language.

-- SRX file for the target (Configuration key = targetSrxPath)

Enter the path or URL of the SRX document where to take the SRX rules for the target language. Note that when the same document for both languages (most of the cases), the document is read only once, saving processing time.

-- Verify in-line codes for text units with single segment (Configuration key = checkSingleSegUnit)

Set this option to check the in-line codes of the text units that have only a single segment. If this option is set the utility checks for possible issues and may prompt the user to validate the alignment. If this option is not set the utility assumes aligned text units with a single segment do not need manual validation.

-- Use auto-correction automatically (Configuration key = useAutoCorrection)

Set this option to automatically try to correct text units that have a different number of source and target segments. The user is always prompted to confirm the modifications made by auto-correction.
